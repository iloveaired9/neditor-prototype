# 기능 비교 보고서: CHEditor vs. Neditor

이 보고서는 기존 `cheditor` 기능과 현재 개발 중인 `neditor` 구현을 비교하여, 부족한 기능을 파악하고 개선 기회를 식별하기 위해 작성되었습니다.

## 비교 표

| 기능 카테고리 | CHEditor 상태 | Neditor 상태 | 비고 |
| :--- | :--- | :--- | :--- |
| **기본 서식** | 높음 (완성) | 중간 (필수 위주) | Neditor는 현대적인 프리셋에 집중하며, CHEditor는 레거시 옵션(첨자 등)이 더 많음. |
| **이미지 즉시 붙여넣기** | 기본 수준 | 높음 (최적화) | Neditor v1.1.1 구현이 더 빠르고 현대적임. |
| **표(Table) 관리** | 높음 (복잡) | 중간 (기본) | CHEditor는 셀 속성 편집이 가능하며, Neditor는 빠른 삽입과 기본 수정을 중시함. |
| **링크 스크랩** | 수동/스크립트 | 높음 (AI 대응) | Neditor의 ScrapPlugin이 시각적으로 더 프리미엄하며 견고함. |
| **미디어 임베드** | 플래시/미디어 | 유튜브 특화 | Neditor는 전용 유튜브 플러그인이 있으며, CHEditor는 범용적이지만 낡은 방식(Flash). |
| **뽐뿌 동기화** | 기본 내장 | 기본 내장 | 두 에디터 모두 하단 첨부 리스트 동기화를 지원함. |
| **AI 편집** | 지원 안 함 | 높음 (핵심 기능) | Neditor만의 고유 기능. |
| **UI/UX 디자인** | 레거시 (90년대) | 프리미엄 (현대적) | Neditor는 글래스모피즘과 현대적인 CSS를 사용함. |

## 기능 격차 (Neditor 추가 권장 사항)

### 1. 서식 기능 강화
- **위 첨자 및 아래 첨자**: 기술 문서 작성 시 유용함.
- **줄 간격**: 가독성 제어를 위해 필요함.
- **단락 스타일 (H1, H2)**: 현재 Neditor는 글자 크기로 계층을 구분하나, 시맨틱한 제목 태그 추가가 필요함.

### 2. 특수 객체
- **특수 문자 (Symbol)**: 기호 선택 팝업은 사용자 요청이 빈번한 기능임.
- **이모티콘 (Smiley)**: 표준 이모지 지원 또는 전용 아이콘 선택기.
- **텍스트 상자**: 단순 인용구를 넘어선 스타일링된 "공지" 또는 "강조" 블록.

### 3. 고급 표 편집
- **셀 병합/분할**: CHEditor는 전용 대화상자를 통해 지원함. Neditor에도 점진적 도입 필요.
- **표 속성**: 셀별 테두리 색상, 여백, 배경색 설정 기능.

### 4. 일반 삽입 기능
- **가로줄 (HR)**: 단순하지만 필수적인 구분선.
- **범용 임베드**: 외부 서비스의 iframe/script 태그를 안전하게 붙여넣는 방식 (미디어 임베더).

## 종합 권장 사항
Neditor는 이미 **시각적 품질**, **AI 통합**, **현대적 워크플로우**(즉시 붙여넣기, 스크랩 카드) 측면에서 CHEditor를 능가했습니다. 다음 단계에서는 CHEditor의 구조적 편집 기능(제목 태그, 가로줄)과 특화된 삽입 도구(기호, 이모티콘)를 이식하여 "기능적 완결성"을 확보하는 데 집중해야 합니다.
