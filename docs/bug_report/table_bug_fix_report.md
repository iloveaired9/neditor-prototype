# 테이블 컬럼 생성 버그 분석 및 수정 보고서

## 1. 버그 개요
- **현상:** 테이블 셀 내에 탭(`\t`) 문자가 포함된 텍스트를 붙여넣을 때, 의도치 않게 새로운 컬럼(또는 중첩된 테이블 구조)이 생성되는 문제.
- **영향:** 사용자 의도와 다르게 표의 레이아웃이 깨지고 데이터 입력이 불편해짐.

## 2. 원인 분석
- **브라우저 기본 동작:** HTML5의 `contenteditable` 속성이 적용된 영역에서는 브라우저마다 텍스트 붙여넣기 시 고유의 처리 로직을 가짐. 
- **탭 문자 처리:** 특히 크롬(Chrome) 및 현대적인 브라우저들은 탭(`\t`)으로 구분된 텍스트를 "스프레드시트 데이터" 또는 "테이블 데이터"로 간주함.
- **결과:** 사용자가 탭이 포함된 텍스트를 붙여넣으면 브라우저가 이를 자동으로 해석하여 새로운 `<td>` 또는 새로운 `<table>` 구조를 삽입하려고 시도하면서 컬럼이 늘어나는 현상이 발생함.

## 3. 해결 방안 (수정 내용)
### 3.1 수정 전략
- 브라우저의 자동 해석이 일어나기 전(Paste 이벤트 단계)에 탭 문자를 안전한 공백 문자로 치환하여 삽입.

### 3.2 코드 수정 내역
- **대상 파일:** `js/core/Neditor.js`
- **수정 로직:**
  1. 에디터 영역(`this.el`)에 글로벌 `paste` 이벤트 리스너 추가.
  2. 클립보드 데이터에서 텍스트를 추출.
  3. 텍스트에 탭(`\t`) 문자가 포함되어 있는지 확인.
  4. 포함되어 있다면 `e.preventDefault()`를 호출하여 브라우저의 기본 붙여넣기 동작을 차단.
  5. 모든 탭(`\t`) 문자를 4개의 공백('    ')으로 치환.
  6. `document.execCommand('insertText', ...)`를 사용하여 치환된 텍스트를 현재 커서 위치에 삽입.

```javascript
// js/core/Neditor.js 수정 부분
this.el.addEventListener('paste', (e) => {
    if (e.defaultPrevented) return;

    const text = (e.clipboardData || window.clipboardData).getData('text');
    if (text && text.includes('\t')) {
        e.preventDefault();
        // 탭을 4개의 공백으로 변환하여 컬럼 분리 방지
        const cleanText = text.replace(/\t/g, '    ');
        this.execCommand('insertText', cleanText);
    }
});
```

## 4. 검증 결과
- **테스트 시나리오:** 2행 1열 테이블 생성 후, 탭이 포함된 텍스트 붙여넣기.
- **결과:** 컬럼 증가 없이 하나의 셀 안에 탭이 공백으로 치환되어 정상적으로 입력됨을 확인.

## 5. 결론
이번 수정을 통해 탭 문자로 인한 테이블 레이아웃 깨짐 현상을 근본적으로 차단하였으며, 사용자는 엑셀이나 다른 문서에서 복사해온 데이터를 테이블 내에 안전하게 붙여넣을 수 있게 되었습니다.
